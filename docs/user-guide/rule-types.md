# 规则类型

RuleForge 支持以下规则类型：

## 向导式规则集

通过可视化界面配置的规则集合。每条规则由条件（IF）和动作（THEN）组成，无需编写代码。适合业务人员直接维护。

**适用场景**: 标准审批规则、分级判定逻辑。

## 脚本式规则集 (UL)

使用脚本语言编写的规则集，灵活度更高，支持复杂逻辑表达式。

**适用场景**: 需要复杂计算或条件组合的规则。

## 决策表

以表格形式组织的规则，行表示规则，列表示条件或动作。支持范围条件、枚举条件等。

**适用场景**: 多条件组合判定，如根据年龄、收入、信用评分等维度组合出不同结果。

## 脚本决策表

结合脚本表达式的决策表，单元格中可使用脚本语言编写复杂条件。

**适用场景**: 决策表中需要复杂计算逻辑的场景。

## 决策树

以树形结构组织的决策逻辑，从根节点出发，根据条件分支到达叶子节点得出结论。

**适用场景**: 层层递进的分类决策，如风险等级划分。

## 评分卡

为各项指标分配权重和分数，汇总得出总分，再根据总分区间映射结果。

**适用场景**: 信用评分、综合评估。

## 决策流

使用 Flowable BPM 引擎编排多个规则节点的执行顺序，支持并行、串行、条件分支等流程控制。

**适用场景**: 复杂的多步骤决策流程，需要编排多种规则类型的场景。
